# Fraud Monitoring Dashboards

> **Live demo:** [nockbarry.github.io/fraudMonitoring](https://nockbarry.github.io/fraudMonitoring/)

Four companion dashboards designed around the failure modes that matter in
production fraud modeling. Each is a single self-contained HTML file using
Plotly via CDN — no build step, no server. The four hubs cross-link to each
other from the sidebar of every page.

## Dashboards

- **[Model Performance Hub](https://nockbarry.github.io/fraudMonitoring/model_performance_hub_demo.html)** — ROC, PR, calibration, lift & gains, daily stability, feature importance, model comparison, cohorts, operating thresholds.
- **[Feature Analysis Hub](https://nockbarry.github.io/fraudMonitoring/feature_analysis_hub_demo.html)** — per-feature importance, univariate AUC, drift, missing rate, behavior buckets, partial dependence.
- **[Stability & Calibration Hub](https://nockbarry.github.io/fraudMonitoring/stability_monitoring_hub_demo.html)** — diagnostic toolkit for the failure mode where AUC stays flat but production thresholds break: importance concentration, conditional PSI, adversarial validation, decile drift, dual-signal check, threshold construction, post-deploy monitoring.
- **[Fraud Rings Hub](https://nockbarry.github.io/fraudMonitoring/fraud_rings_hub.html)** — network-driven view of coordinated fraud activity across 10 archetypes (synthetic-ID, mule, card-testing, bust-out, ATO, triangulation, refund, first-party, friendly, promo abuse). Catalog, network graph, ring focus, archetype breakdown, model coverage, feature signals, transaction status, cross-ring overlap, growth & decay, detection methods, impact, investigation queue.

## Synthetic data

Every dashboard ships with a deterministic synthetic dataset (seeded PRNG so
renders are identical across reloads):

- **10 weeks** of transaction-level data
- **630k transactions** generated by an entity-driven model (accounts → cards →
  merchants → devices → IPs) with proper time-windowed velocity features
- **84 engineered features** spanning invariant (device/behavioral/structural),
  tactical (current attack surface), and compositional (aggregations / embeddings)
  buckets
- **2 trained models** — GBM champion and LR challenger — across train / cal /
  oot / recent splits, with a planted regime shift at week 8
- **243 fraud rings** across 10 archetypes plus a non-ring fraud track for
  uncoordinated single-actor activity

## Building from the Python pipeline

```bash
cd builder
python demo_fraud.py            # generates dataset (cached) + trains models + emits all 3 *_demo.html files
python demo_fraud.py --regen    # force dataset regeneration
```

Cold run (dataset gen + training + 3 reports): ~133s. Hot run (cached dataset): ~33s.

## Notes

- All HTML files are standalone — open directly in a browser, no server needed.
- Plotly is loaded from CDN.
- The original `*.html` versions are hand-authored synthetic demos kept for
  reference; the `*_demo.html` versions are populated by the Python builder
  from real model outputs and cohort breakdowns.
- The fraud rings hub is currently synthetic-only (planned: a Python builder
  to drive it from a real entity-graph dataset).
