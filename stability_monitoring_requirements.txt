● Stability & Calibration Monitoring Hub — Requirements

  0. Global / framework

  - G1. Single standalone HTML file. No external assets except Plotly via CDN (plotly-2.35.0.min.js).
  - G2. Self-contained synthetic data; deterministic across reloads. Use a seeded PRNG (mulberry32) and a Box-Muller gauss()
  helper. Single seed constant at top of script.
  - G3. All charts use a shared baseLayout() with: Inter font stack 11px, white paper/plot background, horizontal legend below
  plot at y=−0.18, common margins.
  - G4. Common color tokens: invariant=#0ea5e9, tactical=#f59e0b, compositional=#a855f7, accent=#6366f1, good=#10b981,
  warn=#f59e0b, bad=#ef4444, train=#64748b, cal=#0ea5e9, oot=#7c3aed, recent=#ef4444.
  - G5. Hash-based router: #/<section> or #/<section>/<sub>. Back/forward navigation must work via hashchange.
  - G6. On every navigation: purge existing Plotly elements, clear #main, re-render page from a route table.
  - G7. Helper functions required: el(tag, cls, html), pageHeader(title, desc), chartCard(title, sub, height),
  baseLayout(extra), fmtPct(v, d), fmt(v, d), makeCallout(title, body, kind).
  - G8. Callout kinds: info (indigo), good (green), warn (amber), bad (red).
  - G9. Pill kinds: pill-good, pill-warn, pill-bad, pill-accent, pill-inv, pill-tac, pill-cmp.

  1. Layout / chrome

  - L1. Two-pane app: 280px dark sidebar (#0f172a), light main content area.
  - L2. Sidebar contains: hub title with accent square, sub-label, two cross-hub links (Model Performance Hub, Feature Analysis
   Hub), nav tree, footer line.
  - L3. Nav tree supports two item kinds: direct (one click navigates) and section with children. Sections collapse/expand on
  click; chevron rotates.
  - L4. Optional badges on nav items (e.g. AUC 0.78 warn, live accent).
  - L5. Topbar (56px) with breadcrumbs on the left, comma-separated topstat strip on the right showing OOT AUC, ECE, ADV-AUC,
  maturity-day count.
  - L6. Breadcrumbs format: Stability & Calibration › <Section> › <Subsection>. Last crumb bolded.
  - L7. Page header pattern: 20px bold title + 13px description paragraph (max 880px wide, line-height 1.55).
  - L8. KPI grid: 4 columns at desktop, 2 at <1100px, with optional left border accent (accent/good/warn/bad).
  - L9. Chart card pattern: bordered white card, head with title + optional subtitle, plot div with configurable height
  (default 360px).
  - L10. Tables: .rt-tbl style — uppercase 11px column headers, tabular-nums in cells, .hl class for emphasized rows.

  2. Synthetic data model

  - D1. Timeline: 120 daily observations starting 2026-01-01, formatted as ISO date strings.
  - D2. Regime shift: feature/score behavior changes starting at day index 70, ramping to full effect by ~day 95.
  - D3. Feature catalog (≥150 features):
    - 20 hand-named features with realistic fraud-domain names (e.g. device_entropy_24h, velocity_amt_card_1h,
  embed_merchant_d8).
    - ~130 padded feat_NNN features with exponentially decaying gain.
    - Each feature has: name, bucket (invariant/tactical/compositional), gain, stable_target (bool), drift_psi.
    - Gains renormalize to sum to 1; sorted descending.
  - D4. Daily series per day: fraud_rate, mean_score, auc, ece, top_decile fraud rate. AUC stays in 0.90–0.93 band; ECE
  quadruples post-shift; mean score lifts only partially relative to fraud rate.
  - D5. Adversarial validation block: scalar ADV_AUC=0.781 and a list of ~10 top discriminator features with name, imp, note.
  Notes encode bucket via substring (invariant/tactical/compositional).
  - D6. Decile fraud-rate matrix: 10 deciles × 120 days. Baseline rate per decile increasing geometrically. Top deciles
  (D8–D10) drift up post-day-70.
  - D7. Score histogram generator producing 30 bins on [0, 1] for three populations:
    - Training: high separation factor, no shift, no noise → bimodal.
    - Calibration: moderate separation, small right shift, light noise.
    - Recent week: moderate separation, larger right shift, more noise.
    - Plus a "deployed" population for evaluating bin edges against reality.
  - D8. Decile bin-edge function: produces 11 quantile-based edges from a histogram.
  - D9. Apply-edges function: returns per-bin population share when arbitrary edges are evaluated against an arbitrary
  histogram.
  - D10. Weekly bin-population matrices: 12 weeks × 10 bins, two versions (raw-score-binned drifts post-week-6, isotonic-binned
   much tighter).
  - D11. Cohort table: ~7 cohorts each with name, n, train_fraud, oot_fraud, train_auc, oot_auc, fcr_at_2pct. At least one
  cohort must show OOT-AUC collapse (Δ > 0.10) hidden inside healthy aggregate.
  - D12. Maturation curve: 91-day Weibull-shaped CDF reaching 95% at 40 days; expose idx95 and 50/99% lookups.
  - D13. Apparent-vs-true fraud rate by lookback: true_fr constant, apparent = true_fr * matureCDF[d].
  - D14. Conditional PSI table: ~15 features with marg, cond, bucket, verdict ∈ {real shift, partial, mix change, minor,
  stable}.
  - D15. Post-deploy monitoring: 60-day series of inputDrift and conceptDrift; concept drift inflects later but exceeds input
  drift.

  4. Interactions

  - I1. Click any nav item → route changes, page renders, hash updates.
  - I2. Section heads in the nav toggle collapsed state via class toggle, persisted in collapsedSections Set for the lifetime
  of the page.
  - I3. Active nav item gets .active class with accent left border + tinted background; updated on every render.
  - I4. Browser back/forward triggers hashchange → route resolves and renders without full reload.
  - I5. Cross-hub links in the sidebar are plain <a href> to the other two HTML files in the directory.
  - I6. Plotly charts are responsive (responsive: true, no modebar).
  - I7. Hover tooltips on chart traces show formatted values; PSI scatter additionally specifies a hovertemplate showing
  feature name and both PSI values.
  - I8. No client-side state beyond current path and collapsed sections — everything else is derived from the synthetic data
  block.

  5. Acceptance criteria

  - A1. File loads in a modern browser with no console errors.
  - A2. All 16 pages render distinct content with at least one chart or table each.
  - A3. AUC line on the overview chart visibly stays within ~0.01 of its baseline while ECE roughly quadruples — i.e. the
  headline narrative is visible at a glance.
  - A4. "New merchant (<30d)" cohort row is highlighted in the cohorts table and visibly an outlier on the train-vs-OOT-AUC
  scatter.
  - A5. Bin-volume drift chart shows the training-edges strategy is the worst by Σ|bin% − 10%|, calibration-edges is best.
  - A6. Isotonic-then-bin subplot shows visibly tighter per-bin trajectories than the raw-score subplot post-week-6.
  - A7. Conditional PSI scatter shows several tactical features in the "mix change" zone (high marginal, low conditional) and
  at least one in the "real shift" zone (high marginal, high conditional).
  - A8. Adversarial-validation top-10 contains tactical and compositional features near the top, with session_keystroke_var
  (invariant) clearly at the bottom.
  - A9. Determinism: two reloads produce visually identical charts.
